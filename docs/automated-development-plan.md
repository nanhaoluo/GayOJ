# gayoj 自动化开发任务计划书与提示词

> 来源文档：`1.md`  
> 适用目标：让自动化开发代理按阶段持续填充 gayoj，从当前 FastAPI + Vue MVP 演进到设计文档定义的 v1.0 系统。  
> 关键边界：代码题只允许在线评测；客观题允许在线即时判分与离线 CLI 训练；普通题目详情不得泄露客观题 `judge_config`。

## 1. 项目目标

gayoj 是覆盖“选手训练、教练管理、裁判执法、系统运维”的综合算法训练平台。v1.0 需要支持：

- 用户、角色、权限、账号安全。
- 代码题在线评测：提交队列、沙箱执行、结果回写、评测节点调度。
- 客观题评判：填空题、单选题、多选题在线即时判分。
- 客观题离线训练：签名离线包、本地判分、联网后同步。
- 题库管理：CRUD、标签、知识点、测试数据、导入导出、题单/试卷。
- 比赛系统：ACM/OI/IOI/CF，多赛制榜单，报名与队伍、Clarification、封榜、代码打印、气球、重测、虚拟参赛。
- 多角色端：选手、教练、裁判、管理员。
- 社交与通知：讨论、题解、评论、站内通知、实时推送。
- 统计分析与开放平台：用户画像、题目统计、OpenAPI、Webhook、SDK。
- 部署运维：Docker Compose、生产 K8s、监控、日志、备份、安全审计。

## 2. 当前基线

当前仓库已有可运行 MVP：

- 后端：`apps/api`，FastAPI，MySQL 主存储 + SQLite 兜底快照存储。
- 前端：`apps/web`，Vue 3 + TypeScript + Vite。
- CLI：`tools/offline-cli/gayoj_offline.py`，支持客观题离线包、练习结果同步。
- 已有功能：登录、题库、提交、题单、比赛、Clarification、讨论、通知、教练端、裁判端、管理端、系统配置。

后续自动化开发应优先保持现有技术栈和目录结构，逐步完善业务和提高可靠性。

## 当前存储基线（MySQL + SQLite）

- 运行时存储已改为 `apps/api/storage` 中央数据库层：MySQL 为主存储，SQLite 为 MySQL 不可用时的兜底存储。
- `apps/api/storage/dev-db.json` 只作为兼容种子快照导入数据库，不再作为运行时 JSON 写入存储。
- 业务代码必须通过 `apps/api/app/db` 仓储函数读写数据；数据库连接、SQL、SQLite 缓存和 MySQL 兜底逻辑集中在 `apps/api/storage`。
- 新增存储相关任务必须继续保证：代码题只入在线评测队列或 worker；普通题面接口不暴露客观题 `judge_config`；CLI 只处理填空题、单选题、多选题。
- SQL 编写必须使用参数化查询和固定表名/字段名，避免 SQL 注入、SQL 穿透和动态拼表名。

## 3. 自动化开发总规则

每个自动化任务必须遵守：

1. 先阅读 `1.md`、`README.md`、`docs/mvp-architecture.md` 和相关代码，不允许凭空改架构。
2. 每次只改一个清晰模块，避免跨模块无边界重构。
3. 代码题不得在 Web、API 或 CLI 本地执行用户代码，必须进入在线评测队列或评测 worker。
4. 客观题 `judge_config` 只允许管理端、服务端规则引擎和授权离线包读取，普通题面接口不得返回答案。
5. 所有新增 API 必须以 `/api/v1` 为前缀，返回结构保持 JSON 可序列化。
6. 所有涉及用户输入的接口必须校验权限、参数和资源归属。
7. 前端新增页面必须响应式，移动端可用，不能只做桌面宽屏。
8. 每个任务完成后必须运行最小验证：后端编译、前端构建、关键接口请求或端到端脚本。
9. 不得破坏现有演示账号：`alice / coach / judge / admin`，密码 `gayoj123`。
10. 遇到存储结构变化，必须写兼容迁移，不能要求删除 `apps/api/storage/dev-db.json`。

Phase 5 完成标准：

- 在线与离线对同一份客观题规则得出一致结果，填空题、单选题、多选题的边界用例都稳定。
- 离线包只包含授权客观题，不包含代码题；离线包必须签名、可过期、可追踪来源。
- CLI 只负责客观题练习、结果导入和同步，不编译、不运行、不评判代码题。
- 离线结果同步必须幂等，重复同步不重复计分，冲突有可审计的拒绝原因。
- 授权人员可以查看逐项得分、答卷历史和复盘信息，普通题面和普通用户始终看不到标准答案。
- 离线训练支持断点续练、答案文件导入和结果回看，过期包与无授权包必须失效。

Phase 6 完成标准：

- 训练赛：管理员或教练可快速创建比赛，配置题目、时间、赛制、可见性和补题规则；选手可正式参赛、虚拟参赛、查看个人提交和赛后复盘。
- 正式赛：支持报名名单锁定、队伍/个人参赛、封榜、Clarification、代码打印、气球、裁判工作台、重测、榜单导出、纯净展示页和最终归档。

## 7. 自动化执行节奏

建议每个自动化开发循环采用固定节奏：

1. 选择 1 个任务 ID。
2. 由总控代理拆成后端、前端、测试、文档子任务。
3. 子代理各自完成不重叠文件范围。
4. 总控代理集成并解决冲突。
5. 运行验证命令。
6. 写入简短变更记录。
7. 进入下一任务。

每轮最大改动建议：

- 后端：不超过 6 个核心文件。
- 前端：不超过 4 个页面或 1 个完整业务域。
- 数据迁移：一次只迁移一个领域。
- 评测 worker：一次只实现一种语言或一种状态链路。
